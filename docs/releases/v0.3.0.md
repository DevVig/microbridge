# Microbridge 0.3.0 (2026-07-20)

This release expands host coverage while keeping every integration on a public,
capability-gated contract.

## New

- **Factory:** one-click installation of official lifecycle hooks, official
  JSON-RPC interrupt, and model-aware reasoning-effort control from the Micro
  knob. Existing Factory hooks are preserved and Microbridge leaves no resident
  Factory helper process.
- **Conductor:** Codex and Claude sessions under Conductor workspaces are named
  correctly through the built-in journal watchers with no extra poller or
  network call.
- **Synara:** a first-class adapter card and documentation make its automatic
  Codex/Claude host attribution visible; no pairing code is required.
- **T3 Code:** double-press/open now uses T3 Code's official environment/thread
  deep link instead of keyboard automation.
- **Developer run path:** `./script/build_and_run.sh` builds both sidecars,
  bundles a real macOS app, launches it, and supports verify/debug/log modes.

## Honest capability boundary

- T3 Code reasoning effort remains disabled because the paired snapshot does
  not publish provider/model option descriptors needed to choose the next valid
  value safely.
- Conductor exposes lifecycle attribution, but active-session effort changes
  and a reliable public journal-id-to-Conductor-session-id mapping are not
  available. Microbridge does not read Conductor private state to guess.
- Physical Codex Micro behavior still requires validation on real hardware.

## Verification

- Rust workspace format, strict Clippy, and all workspace tests.
- Cursor hook tests, 9 UI surface tests, production frontend build.
- 3 Tauri integration tests, including preservation/removal of existing Factory
  hooks.
- A real Apple Silicon debug `.app` launch with native `microbridged` and
  `microbridgectl` sidecars present in the bundle.
