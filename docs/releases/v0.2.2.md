# Microbridge 0.2.2 (2026-07-18)

## Highlights

- The macOS app now opens the live device-and-threads popover on left click and reserves the native utility menu for right click.
- The software Agent Keys mirror the daemon's exact LED frame: live thread assignment, effective palette color, brightness, pause state, and focus.
- Clicking a simulated Agent Key focuses its assigned thread through the same daemon route as the physical key; double-click requests host open when the adapter supports it.
- Direct-download apps now bundle and start `microbridged`, so a DMG install works without a separate daemon or Marketplace plugin download.

## Fixes

- Removed the production fallback that silently displayed canned browser-demo threads when the daemon was unavailable.
- Prevented a second daemon from replacing the live daemon's Unix socket.
- Kept new LED-frame and Agent Key activation protocol fields backward compatible with v0.2.1 clients and daemons.

## Upgrade notes

- Direct installs update through **Check for Updates…** or the optional launch check.
- Homebrew installs update with `brew update && brew upgrade microbridge`; the formula continues to own its background service.
- No config migration is required.

## Verification

- Rust workspace tests and Clippy with warnings denied.
- Tauri crate check and Clippy with warnings denied.
- UI component tests and production build.
- Live local run discovered current Codex and Claude sessions, emitted the resolved six-key frame, and focused the exact session assigned to a simulated key.
- A direct-download candidate launched its bundled daemon and served live sessions without Homebrew.

## Risk / notes

- Physical Codex Micro key input remains a hardware validation gate; this patch validates the software frame and activation route.
- The accessory-only menu-bar item could not be enumerated by the desktop automation layer. The native menu is now structurally detached from the tray icon and invoked only from a right-button release; final signed-candidate left/right click validation remains a release gate.
