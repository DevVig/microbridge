# Microbridge 0.2.3 (2026-07-19)

## Highlights

- Agent Keys now default to the focused app, so the deck follows the IDE or
  harness currently in use while still supporting mixed-app and custom layouts.
- Synara, T3 Code, Codex Desktop, and standalone Codex CLI sessions are labeled
  from their real host metadata instead of collapsing into "Codex CLI."
- The menu bar app can start at login from every install channel after a
  one-time prompt, with a matching toggle in Settings.
- The left-click popover now fits the active display, shows up to 10 threads in
  a scrolling list, keeps its footer reachable, and dismisses on outside click.
- Automatic update checks remain opt-in and now run at most once every 24 hours;
  manual checks stay immediate.

## Fixes

- Removed subagent rollout journals from the user-facing session list.
- Removed fabricated demo sessions from the real app when the daemon is offline;
  the app now shows an honest disconnected state.
- Prevented an initial snapshot request from overwriting a newer live bus event.
- Replaced the popover's stacked native and oversized CSS shadows with one
  compact, theme-aware shadow.
- Prevented development builds from registering temporary binaries as login
  items.

## Upgrade notes

- Direct installs update through **Check for Updates…** or the optional daily
  launch check.
- Homebrew installs update with `brew update && brew upgrade microbridge` after
  the v0.2.3 formula PR merges.
- No configuration or data migration is required.

## Known issues

- Physical Codex Micro HID input and LED behavior still require validation with
  the production device. This release improves the software control plane and
  simulator; it does not claim hardware validation.

## Verification

- Rust workspace tests and CI on macOS and Ubuntu.
- Adapter tests for host attribution and subagent filtering.
- UI component tests and production build.
- Native Tauri configuration and compile checks.
- Signed/notarized DMGs, updater artifacts, and Homebrew candidate installs are
  validated by the release workflow before promotion.

## Risk / notes

- No breaking protocol, configuration, or storage changes.
- The release remains a prerelease until the generated Homebrew formula PR is
  merged and clean public-tap installations pass on Apple Silicon and Intel.
